Add New Node to Cluster
Important: Make sure to add the replica to the cluster before starting the new node with the --join flag. Replica ID is required and must be greater than 0 Port is required and must be between 1024 and 65535 Port is automatically calculated as 17000 + Replica ID (but can be edited) Valid IP address is required Next Steps: After successfully adding the replica, start your new node with one of these commands:

Option 1: Same machine (each node on different port):
go run . -self-member-host {{ addNodeForm.get('host')?.value }} -cluster-base-port {{ enhancedClusterInfo?.port_configuration?.cluster_base_port || '17000' }} -replica {{ addNodeForm.get('replica_id')?.value }} -rest-port {{ (enhancedClusterInfo?.port_configuration?.rest_port || 3000) + (addNodeForm.get('replica_id')?.value || 0) }} -grpc-port {{ (enhancedClusterInfo?.port_configuration?.grpc_port || 4000) + (addNodeForm.get('replica_id')?.value || 0) }} -join --role=scheduler,connector

Option 2: Separate machines (can use same ports since they are on different hosts):
go run . -self-member-host {{ addNodeForm.get('host')?.value }} -cluster-base-port {{ enhancedClusterInfo?.port_configuration?.cluster_base_port || '17000' }} -replica {{ addNodeForm.get('replica_id')?.value }} -rest-port {{ enhancedClusterInfo?.port_configuration?.rest_port || '3000' }} -grpc-port {{ enhancedClusterInfo?.port_configuration?.grpc_port || '4000' }} -join --role=scheduler,connector
